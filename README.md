# Parametric STL Generator

Самодостаточный HTML/CSS/JS инструмент для параметрического моделирования куба-контейнера
с экспортом в STL. Открывается прямо в браузере, без сборки и зависимостей — весь код,
**включая встроенную (инлайн) Three.js r128 для 3D-превью и шрифты IBM Plex (WOFF2, latin+cyrillic,
data-URI)**, лежит в одном файле `parametric-stl-generator.html`. Файл ПОЛНОСТЬЮ автономен: при загрузке
не делает НИ ОДНОГО внешнего запроса (проверено), поэтому работает без интернета целиком — и 3D-превью, и
типографика. Размер файла — ~1.3 МБ.

## Возможности

- **Установка как приложение (PWA)**: генератор можно установить на ПК/телефон — иконка на рабочем столе /
  домашнем экране, запуск в отдельном окне без адресной строки, работает офлайн (service worker кэширует
  оболочку). В панели есть кнопка «⤓ Установить приложение» (появляется, когда браузер это поддерживает; на
  iPhone/iPad показывает шаги «Поделиться → На экран „Домой“»). Компаньон-файлы: `manifest.webmanifest`,
  `sw.js`, `index.html`, иконки. **Важно:** браузеры устанавливают PWA только со страницы по HTTPS/HTTP —
  из локального файла (`file://`) установить нельзя (сам файл при этом открывается и работает как обычно).
  Как захостить и установить — раздел [«Установка как приложение»](#установка-как-приложение-pwa) ниже.

- **Живое превью с автопереключением на отложенный режим**: правки параметров/логотипов применяются
  сами (дебаунс ~300 мс), пока полная пересборка меша укладывается в ~0.9 с. Когда пересборка
  ИЗМЕРЕНА тяжёлой, автоперестройка отключается: правки лишь помечают превью устаревшим, в окне 3D
  появляется бейдж «превью устарело — обновить» (тап — пересборка). Экспорт/сохранение/переключение
  моделей всегда достраивают синхронно — файл никогда не отстаёт от панели. Поворот и позиция всегда
  живые (дешёвые пути без пересборки). Тест `test_apply_button.js`.
- **Undo/redo** (кнопки «↶ / ↷» + Ctrl+Z / Ctrl+Shift+Z / Ctrl+Y): до 60 шагов по параметрам,
  логотипам и отверстиям активной модели; история пер-модельная. Тест `test_undo.js`.
- **Сворачиваемый UI**: каждый раздел панели — `<details>` с запоминанием состояния (localStorage);
  группы параметров сворачиваются отдельно; длинные описания спрятаны за «ⓘ справка». На мобильной
  вёрстке между превью и панелью есть ручка-грип — высота 3D-окна регулируется и запоминается.
- **Несколько моделей (сборка)**: карточки моделей — добавить (до 12), дублировать, удалить,
  скрыть/показать, переименовать. У каждой свои параметры, логотипы, отверстия, поворот, позиция и цвет.
  Модели можно **перетаскивать прямо в сцене** (кнопка «✥ перемещение»: драг по модели двигает её по
  площадке с шагом 0.5 мм и делает активной; пустое место по-прежнему вращает камеру). Все ВИДИМЫЕ
  модели выгружаются в один STL (`mergedExportTris`) или в **3MF раздельными объектами** (см. экспорт).
  Булева объединения нет осознанно: слайсеры корректно объединяют пересекающиеся оболочки. Тест
  `test_multi_model.js`.
- **Крышки** (селект «Крышка: …» + кнопка «+ Крышка»): **накидная** (колпак внахлёст с печатным
  зазором), **с защёлками** (бампы 0.7 мм на стенках контейнера + ответные окна в юбке крышки — в
  закрытом положении окно ложится ровно на бамп, тест `test_clips.js`) и **откидная с петлёй**
  (проушины на задней стенке + пластина с ответной проушиной, каналы Ø2 соосны — ось из кусочка
  филамента 1.75; тест `test_hinge.js`).
- **Отверстия**: по нормали выбранной грани, формы — **круг**, **скруглённый прямоугольник**
  (USB-C/щель/паз) и **произвольный SVG-контур** (файл растеризуется в полярную таблицу радиусов и
  режется той же водонепроницаемой машинерией; тест `test_svg_hole.js`). Для круглых на сплошном кубе —
  **потай** под винт («зенковка»/«цекование»), для всех сквозных — **фаска кромок 45°** (`edgeCh`).
  **Пресеты разъёмов**: USB-A/C, micro-USB, HDMI, jack 3.5, DC barrel, SD, microSD, «Вентиляция»
  (щель сразу с массивом). **Массивы**: сетка n×m с шагом или n копий по кругу — и для сквозных, и
  для портов (тест `test_hole_patterns.js`). На СПЛОШНОМ кубе отверстие идёт насквозь (плоский и
  скруглённый по углам куб). На ПОЛОМ контейнере отверстие пробивает ОДНУ стенку в полость (порт);
  работает на плоском полом и на squircle-контейнере, в т.ч. со скруглённым дном, — на кривой стенке
  окно РЕАЛЬНО скруглённое (аннулус «блок → скруглённое окно», тест `test_squircle_rport.js`), и
  портов может быть несколько на разных стенках сразу. Водонепроницаемо БЕЗ булева движка
  (грид-блок + аннулус + общий по углам ствол, `buildBoxWithHoles`, тест `test_holes.js`).
- **Логотипы/рельеф на гранях**: до 18 штук, источники — **картинка, текст, QR-код и процедурные
  текстуры**. QR — встроенный энкодер (byte-режим, версии 1–6, ECC M, валидность доказана
  мини-декодером в `test_qr.js`). Текстуры: рифление верт./гориз., точки, решётка из треугольных
  сот (`test_textures.js`). У логотипа есть **замок пропорций** («Связать Ш/В») по соотношению
  сторон исходника. Кнопка **«3D»** на карточке превращает силуэт (по порогу яркости) в ОТДЕЛЬНУЮ
  экструдированную модель сборки — жетон/брелок (`test_logo3d.js`). Зональная сетка: высокая
  детализация (до 500) тратит треугольники только под логотипом. Совместимы со скруглениями,
  наклоном, squircle (рельеф оборачивается по кривизне), полым контейнером (стенки, дно, пол полости).
- **Два режима куба**: сплошной и **полый контейнер** (открытый верх, тонкие стенки). У полого есть
  **«Глубина полости»**: 0 — полость до дна, больше 0 — лоток с карманом заданной глубины (бывший
  режим «борт» — слит сюда, старые JSON-конфиги мигрируются автоматически).
- **Наклон стенок** (8 независимых углов) — работает во всех режимах.
- **Скругление углов** на сплошном кубе — общий радиус ИЛИ раздельно по 3 группам рёбер
  (верх/низ/вертикали): 12-радиусный водонепроницаемый генератор (`buildAsymRoundedBox`, разбор в
  `NOTES.md`). Полый контейнер скругляется целиком (внешние + внутренние рёбра, острый или закатанный
  кант). Совместимо с логотипами (тесты `test_fillet_logo*.js`, `test_rounded_fillet.js`).
- **Squircle / супереллипс**: сечение-супереллипс (0 — прямоугольник, 100 — эллипс) на сплошном и
  полом; «скругление верха/низа» дают супереллипсоид/куполообразное дно. Логотипы embossed прямо на
  кривизну, порты со скруглёнными углами — см. отверстия. Тест `test_squircle.js`.
- **Скругление стенок** — знаковый выгиб грани куполом («бочка»/«подушка»), совместим с логотипами,
  наклоном и скруглением углов; на полом полость повторяет выгиб. Тест `test_wall_bulge.js`.
- **Сетчатое ромбовидное дно** полого контейнера: шаг ромба, толщина ребра, сплошной кант,
  скруглённый профиль ребра и СВОЙ слайдер «Сетка: детализация дна» (10…500). Совместимо с
  логотипами (на дне полости рельеф «висит» на фоне решётки) и с перегородками. Тест
  `test_lattice_floor.js`.
- **Фаска верхнего внешнего канта** полого контейнера (ширина + угол, `test_chamfer.js`).
- **Штабелируемость**: ножки-фиксаторы под дном, входящие в полость такого же бокса снизу
  (`test_stack.js`).
- **Перегородки-органайзер**: divX×divZ отсеков в полом/лотке, высота в %, толщина; на squircle
  длина режется по хорде; работают и поверх сетчатого дна (`test_dividers.js`).
- **Gridfinity-бины**: ячейки X/Z (×42 мм), высота в юнитах (×7 мм), ноги с ОФИЦИАЛЬНЫМ профилем
  сокета (тест держит инвариант вложения в огибающую), **стекинг-губа** (бины ставятся друг на друга
  и самоцентрируются) и **магнитные карманы Ø6.5×2.4** (4 на ячейку, ±13 мм). Логотипы/отверстия/
  перегородки/сетка работают на теле бина. Тест `test_gridfinity.js`.
- **Печатаемость**: в сводке — точный объём детали, вес (PLA/PETG), оценка времени печати,
  **площадь нависаний >45°** («нужны поддержки») и предупреждения о тонких элементах; кнопка
  **«⟳ Автоориентация»** перебирает 6 ориентаций и ставит модель с минимумом поддержек
  (`test_print_check.js`).
- **Неблокирующая проверка герметичности** в Web Worker (потолок 600k треугольников) + финальная
  сварка `snapWeldTris` на выходе каждого билда (кластеризация вершин 5e-5 мм — микрощели от двойных
  float-путей исключены).
- **Экспорт**: STL (Binary, вся сборка одним файлом) и **3MF** (каждая видимая модель — отдельный
  объект; свой OPC/zip-писатель, `test_3mf.js`). **Конфигурация сборки (JSON)**: сохранение/загрузка
  всех моделей с параметрами, логотипами (через встроенное превью), отверстиями, позициями и цветами.

## Структура

Всё лежит в корне репозитория (плоская раскладка, без подкаталогов):

```
parametric-stl-generator.html   — весь инструмент (открыть в браузере)
index.html                      — точка входа для хостинга (редирект на генератор; корень GitHub Pages)
manifest.webmanifest            — манифест PWA (имя, иконки, standalone)
sw.js                           — service worker (офлайн-кэш оболочки)
icon-192.png / icon-512.png / icon.svg / apple-touch-icon.png — иконки приложения
run-all.sh                      — запускает весь набор тестов одной командой
PROGRESS.md                     — журнал разработки: что сделано и почему, по каждому проходу
NOTES.md                        — разбор 12-радиусного скругления и доказательство герметичности
asym-12-radius.js               — автономный порт 12-радиусного билдера (источник in-page версии)
stub_preamble.js                — DOM/rAF-стабы, чтобы код страницы запускался под Node
test_*.js, test-*.js            — Node-тесты геометрии и управляющего потока (см. ниже)
minimal-repro-missing-breakpoint.js — минимальный repro одного бага сшивки грани (для справки)
```

## Установка как приложение (PWA)

Генератор — устанавливаемое PWA: иконка на рабочем столе / домашнем экране, запуск в своём окне, офлайн.
Браузеры устанавливают PWA **только со страницы по HTTP/HTTPS** — из локального файла (`file://`) установка
невозможна (файл при этом просто открывается и работает). Значит, сначала инструмент нужно **захостить**.

**Хостинг через GitHub Pages (бесплатно, один раз):** Settings репозитория → Pages → Source: «Deploy from a
branch» → Branch: `main`, папка `/ (root)` → Save. Через минуту заработает адрес вида
`https://<owner>.github.io/<repo>/` (`index.html` перенаправит на генератор). Можно хостить и на любом
статическом сервере — достаточно выложить все файлы из корня рядом.

**Установка с этого адреса:**
- **ПК (Chrome/Edge):** значок установки в адресной строке, либо кнопка «⤓ Установить приложение» в панели,
  либо меню ⋮ → «Установить …».
- **Android (Chrome):** кнопка «⤓ Установить приложение», либо меню ⋮ → «Установить приложение».
- **iPhone/iPad (Safari):** «Поделиться» ⎋ → «На экран „Домой“» → «Добавить» (в панели есть подсказка).

После установки приложение открывается в отдельном окне и работает без интернета (весь код, включая
Three.js и шрифты, уже внутри; service worker кэширует оболочку).

## Тесты

Вся геометрия (углы куба, швы между гранями, водонепроницаемость меша) проверяется в
Node.js перед тем, как попасть в файл — это соглашение действует для каждой фичи в этом
проекте, не только оформлено постфактум.

```
./run-all.sh
```

запускает все наборы тестов (**933 проверки в 34 файлах**) и выводит per-file и общий
итог (ненулевой код возврата при любом падении). Большинству тестов нужен реальный код
из `parametric-stl-generator.html` (не переписанная копия) — `run-all.sh` сам вытаскивает
`<script>`, подставляет DOM-стабы (`stub_preamble.js`), а для `test_debounce_flow.js` и
`test_apply_button.js` ещё и подменяет `regenerate()`/`applyRotationOnly()` на счётчики. Ручная
процедура для каждого файла описана в его собственном заголовке, если нужно запустить один тест отдельно.

`test_logo_zone.js` — единственный самодостаточный тест (не требует извлечения кода из
HTML, содержит собственные копии проверяемых функций): `node test_logo_zone.js`.

## Известные ограничения

- **Полый контейнер (и лоток — «глубина полости» > 0)** полностью скругляются: внешние (низ+вертикали+верхний
  кант `filletTop`) и внутренние полости/кармана (дно+вертикали+кант `filletInnerLip`) рёбра —
  водонепроницаемо. Трей = тот же билдер с мелким карманом (`pocketDepth=rimHeight`). Скругление
  **совместимо с логотипами** (рельеф на внешних стенках + дне полости/кармана, вписан в плоскую зону;
  тест `test_fillet_logo_hollow.js`).
- **Верхний кант**: `filletInnerLip>0` — весь верх закатан (скруглённый внешний `filletTop` +
  скруглённый внутренний кант); `filletInnerLip=0` — весь верх **острый плоский**: обе оболочки со
  скруглёнными вертикальными углами, но острым плоским ободом, без конусов-«шпилей» и без выступающих
  «рожков» на внешних углах (билдер `buildSharpRimHollow`: обе оболочки поднять на радиус вертикали и
  срезать по плоскости проёма; борт = явное кольцо между граничными петлями оболочек). Клампы для
  валидности: внутренняя вертикаль ≤ глубина полости, внешняя вертикаль ≤ внутр.+стенка. Любой суб-0.1
  мм радиус снапится в острое ребро. Пути покрыты `test_rounded_fillet.js`; Node-стресс острого
  билдера — 2000/2000 водонепроницаемы и ориентационно-согласованы.
- Зональная сетка для логотипов есть во ВСЕХ режимах (сплошной, скруглённый и ПЛОСКИЙ полый/лоток):
  оболочка грубая, мелкая детализация — только под логотипом, поэтому высокую детализацию логотипа можно
  крутить без перегенерации всей оболочки. Плотность зон задаёт настройка «Детализация логотипов» (`logoResolution`); у сетчатого дна — свой слайдер «Сетка: детализация дна». Плоский полый без логотипов остаётся на равномерной сетке.
- **Сетчатое диагональное дно** — только для ПЛОСКОГО полого контейнера, и это осознанное решение, а не
  недоделка: решётка со СКВОЗНЫМИ отверстиями осмысленна лишь там, где под дном пустота. У лотка («глубина полости» > 0) под
  карманом сплошная база (сквозному отверстию некуда вести), у скруглённого полого дно — купол (ровная
  диагональная решётка по кривизне определена плохо). **Профиль ребра**: плоский ИЛИ **скруглённый**
  (флажок «Сетка: скруглённый профиль ребра») — верх ребра куполится к отверстиям. Скругление сделано
  как поле высот по ВЕРШИНАМ сетки (дистанция до ближайшей дыры, BFS): соседние ячейки читают общую высоту
  на общем ребре, поэтому шва нет — герметично (пер-ячейковые инсеты рвали бы швы; проверено `test_lattice_floor.js`).
  **Логотипы совместимы с решёткой**: на стенках — как обычно; логотип на дне полости («−Y изнутри»)
  тоже рисуется — сплошной становится только САМА ФОРМА рельефа (по МАСКЕ heightmap>порог, `floorMask`),
  а фон остаётся сеткой, поэтому картинка «висит» на фоне решётки, а не на сплошном квадрате-плитке.
  Форма берётся по маске НЕЗАВИСИМО от глубины, поэтому даже при **«Глубина мм» = 0** логотип виден —
  форма заполняется сплошной заплаткой вровень с дном (плоская вставка); >0 — выпуклый, <0 — гравировка.
  Форс-solid идёт ДО устранения защемлений (любые «шахматки» от тонкой формы вычищаются), высота верха
  эмбоссится ПО ВЕРШИНАМ → герметично, работает и со скруглённым профилем ребра (тест
  `test_lattice_floor.js`). Логотип на внешнем дне («−Y») с решёткой пока не поддержан. Шаг ромба (`cell`) и толщина ребра заданы в мм и
  расцеплены с детализацией стенок (= «Детализация логотипов»): смена размера ромба не огрубляет
  стенки. Диагональ имеет мелкую «лесенку» по краям — чем выше детализация, тем глаже.
- Толщина стенки при наклоне не идеально постоянна по нормали (расхождение небольшое для
  разумных углов, см. `PROGRESS.md`).
- Экстремальный наклон + логотип у самого края сплошного куба может дать негерметичный шов.

Подробная история и обоснование инженерных решений — в `PROGRESS.md`.
