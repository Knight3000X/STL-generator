# Parametric STL Generator

Самодостаточный HTML/CSS/JS инструмент для параметрического моделирования куба-контейнера
с экспортом в STL. Открывается прямо в браузере, без сборки и зависимостей — весь код
(включая Three.js для 3D-превью) лежит в одном файле `parametric-stl-generator.html`.

## Возможности

- **Применение по кнопке «Подтвердить»**: правки параметров и логотипов НЕ пересобирают модель
  на лету — они копятся, а к модели применяются только по нажатию кнопки «Подтвердить» (кнопка
  подсвечивается, пока есть неприменённые изменения). Это убирает тяжёлую перегенерацию на каждое
  нажатие/перетаскивание (был фриз на мобильном). Экспорт STL сначала применяет накопленные правки,
  чтобы файл соответствовал полям. Поворот (rx/ry/rz) остаётся живым (дешёвый путь без пересборки).
- **Три взаимоисключающих режима куба**: сплошной, полый контейнер (открытый верх, тонкие
  стенки), борт/лоток (сплошная основа + неглубокий поддон сверху).
- **Наклон стенок** (8 независимых углов) — работает во всех трёх режимах.
- **Скругление углов** на сплошном кубе — общий радиус ИЛИ **раздельно по 3 группам рёбер**
  (верх / низ / вертикали): если группы различаются, включается водонепроницаемый
  12-радиусный генератор (`buildAsymRoundedBox`, портирован из `asym-12-radius.js`; разбор и
  доказательство — в `NOTES.md`, тесты `test-uniform-and-mixed.js` / `test-3radius-variant.js`).
  Радиусы автоклампятся под размеры, наклон стенок совместим. **Логотипы совместимы со скруглением**
  на сплошном кубе (рельеф на плоских гранях; логотип клампится в плоскую зону, тест `test_fillet_logo.js`).
- **Squircle / супереллипс** (сплошной куб) — горизонтальное сечение становится супереллипсом:
  бока плавно выгибаются, а вертикальные углы скругляются в ЕДИНУЮ гладкую кривую (как иконка iOS),
  вытянутую по высоте с плоскими верхом/низом. Параметр «Squircle» 0..100 %: 0 — прямоугольник,
  ~45 — классический squircle, 100 — эллипс (`x=a·sign(cosθ)·|cosθ|^p`, `buildSquirclePrism`, тест
  `test_squircle.js`). Второй параметр «Squircle: скругление верха/низа» 0..100 % дополнительно скругляет
  верхние/нижние рёбра — получается полный супереллипсоид («надутый куб», `buildSuperellipsoid`); 0 —
  плоские верх/низ. Наклон совместим; отдельно от «скругления стенок» (то — подушка/горб на грани,
  это — единая кривая сечения). **Совместимо с логотипами**: рельеф embossed прямо на изогнутую стенку
  и оборачивается по кривизне (билдеры `buildSquirclePrismLogos`/`buildSuperellipsoidLogos`: поверхность
  строится мелкой сеткой, каждая вершина смещается вдоль локальной нормали; лого клампится от рёбер, так что
  швы сторона↔крышка/полюс остаются закрыты — водонепроницаемо, тест `test_squircle.js`). **Работает во всех
  трёх режимах**: сплошной, полый контейнер (супереллипс-оболочка с тонкими стенками, открытый верх, плоский
  борт) и борт/лоток (супереллипс с неглубоким карманом сверху) — общий билдер `buildSquircleHollow`
  (внешняя стенка + полость `hw−t`/`hd−t` + дно + верхний борт-кольцо). Логотипы садятся на изогнутые внешние
  стенки, дно и пол полости/кармана. В полом/борте вертикальное скругление (squircleV) игнорируется — у
  контейнера плоский борт и он стоит на плоском дне. Перекрывает скругление углов и сетчатое дно.
- **Скругление стенок** — знаковый выгиб грани куполом: положительное значение выгибает наружу
  («бочка»), отрицательное прогибает внутрь («подушка»), 0 — плоско. Купол сходит в 0 к рёбрам грани,
  поэтому меш остаётся водонепроницаемым (тот же механизм смещения, что у логотипов). **Совместимо с
  логотипами** (рельеф логотипа складывается с выгибом грани), с наклоном И **со скруглением углов**
  (при заданных обоих грань выгибается куполом в плоской зоне между скруглёнными рёбрами — инсет-версия
  купола `makeInsetBulgeDisp`). Работает во всех режимах: сплошной куб (все 6 граней), полый контейнер (внешние
  стенки + дно, при этом **внутренняя полость повторяет выгиб — толщина стенки сохраняется**), борт/лоток
  (внешние стенки; лоток сплошной). Тест `test_wall_bulge.js`.
- **Логотипы/рельеф на гранях**: до 18 штук, независимый поворот, автоопределение
  инверсии по прозрачности/яркости изображения. На сплошном кубе — с зональной сеткой
  (детализация только вокруг самого логотипа, не по всей грани). **Совместимо со скруглением**
  во всех режимах: на сплошном кубе, на скруглённом полом контейнере и борте/лотке рельеф ложится
  на плоскую часть грани (внешние стенки, дно/пол полости, грань «−Y изнутри»), вписан между
  скруглёнными рёбрами (тесты `test_fillet_logo.js`, `test_fillet_logo_hollow.js`). **На squircle/
  супереллипсоиде** рельеф embossed прямо на изогнутую стенку и оборачивается по кривизне (лого выбирает
  грань по доминантной оси нормали, смещается вдоль локальной нормали). Наклон стенок совместим (при
  наклон+лого+скругление рельеф слегка тянется наклоном).
- **Сетчатое диагональное дно** полого контейнера (флажок «сетчатое ромбовидное дно») — садовая
  решётка: рёбра ±45°, ромбовидные сквозные отверстия, плоский профиль. Настраиваются шаг ромба,
  толщина ребра, сплошной кант. Водонепроницаемо (строится на общей сетке дна полости, воксельный узор
  с устранением защемлений, тест `test_lattice_floor.js`), совместимо с наклоном стенок и логотипами на
  стенках. Пока только на ПЛОСКОМ полом (не на скруглённом и не на борте).
- Экспорт в STL (Binary/ASCII).

## Структура

Всё лежит в корне репозитория (плоская раскладка, без подкаталогов):

```
parametric-stl-generator.html   — весь инструмент (открыть в браузере)
run-all.sh                      — запускает весь набор тестов одной командой
PROGRESS.md                     — журнал разработки: что сделано и почему, по каждому проходу
NOTES.md                        — разбор 12-радиусного скругления и доказательство герметичности
asym-12-radius.js               — автономный порт 12-радиусного билдера (источник in-page версии)
stub_preamble.js                — DOM/rAF-стабы, чтобы код страницы запускался под Node
test_*.js, test-*.js            — Node-тесты геометрии и управляющего потока (см. ниже)
minimal-repro-missing-breakpoint.js — минимальный repro одного бага сшивки грани (для справки)
```

## Тесты

Вся геометрия (углы куба, швы между гранями, водонепроницаемость меша) проверяется в
Node.js перед тем, как попасть в файл — это соглашение действует для каждой фичи в этом
проекте, не только оформлено постфактум.

```
./run-all.sh
```

запускает все наборы тестов (**383 проверки в 16 файлах**) и выводит per-file и общий
итог (ненулевой код возврата при любом падении). Большинству тестов нужен реальный код
из `parametric-stl-generator.html` (не переписанная копия) — `run-all.sh` сам вытаскивает
`<script>`, подставляет DOM-стабы (`stub_preamble.js`), а для `test_debounce_flow.js` и
`test_apply_button.js` ещё и подменяет `regenerate()`/`applyRotationOnly()` на счётчики. Ручная
процедура для каждого файла описана в его собственном заголовке, если нужно запустить один тест отдельно.

`test_logo_zone.js` — единственный самодостаточный тест (не требует извлечения кода из
HTML, содержит собственные копии проверяемых функций): `node test_logo_zone.js`.

## Известные ограничения

- **Полый контейнер И режим «борт»/лоток** полностью скругляются: внешние (низ+вертикали+верхний
  кант `filletTop`) и внутренние полости/кармана (дно+вертикали+кант `filletInnerLip`) рёбра —
  водонепроницаемо. Трей = тот же билдер с мелким карманом (`pocketDepth=rimHeight`). Скругление
  **совместимо с логотипами** (рельеф на внешних стенках + дне полости/кармана, вписан в плоскую зону;
  тест `test_fillet_logo_hollow.js`).
- **Верхний кант**: `filletInnerLip>0` — весь верх закатан (скруглённый внешний `filletTop` +
  скруглённый внутренний кант); `filletInnerLip=0` — весь верх **острый плоский**: обе оболочки со
  скруглёнными вертикальными углами, но острым плоским ободом, без конусов-«шпилей» и без выступающих
  «рожков» на внешних углах (билдер `buildSharpRimHollow`: обе оболочки поднять на радиус вертикали и
  срезать по плоскости проёма; борт = явное кольцо между граничными петлями оболочек). Клампы для
  валидности: внутренняя вертикаль ≤ глубина полости, внешняя вертикаль ≤ внутр.+стенка. Любой суб-0.1
  мм радиус снапится в острое ребро. Пути покрыты `test_rounded_fillet.js`; Node-стресс острого
  билдера — 2000/2000 водонепроницаемы и ориентационно-согласованы.
- У ПЛОСКОГО (нескруглённого) полого контейнера/борта нет зональной сетки для логотипов — общая,
  более низкая детализация. У СКРУГЛЁННОГО полого/борта зональная сетка есть (как на сплошном кубе).
- **Сетчатое диагональное дно** пока доступно только для ПЛОСКОГО полого контейнера (у скруглённого дно
  строит другой билдер; борт/лоток — на будущее). Профиль ребра плоский (не круглый). Логотипы на самом
  дне не рисуются (перфорация); на стенках работают. Шаг ромба (`cell`) и толщина ребра заданы в мм и
  расцеплены с детализацией стенок (= «Детализация полый контейнер»): смена размера ромба не огрубляет
  стенки. Диагональ имеет мелкую «лесенку» по краям — чем выше детализация, тем глаже.
- Толщина стенки при наклоне не идеально постоянна по нормали (расхождение небольшое для
  разумных углов, см. `PROGRESS.md`).
- Экстремальный наклон + логотип у самого края сплошного куба может дать негерметичный шов.

Подробная история и обоснование инженерных решений — в `PROGRESS.md`.
